🚀 QUICK START GUIDE (5 MINUTES)
================================================================================

STEP 1: Set Up Database (2 minutes)
  ├─ Open SQL Server Management Studio (SSMS)
  ├─ Click File → New → Query with Current Connection
  ├─ Open: Database_Schema_Complete.sql
  ├─ Press F5 to execute entire script
  └─ Wait for completion messages (~5 tables created)

STEP 2: Verify Setup (1 minute)
  ├─ In SSMS left panel, expand "Databases"
  ├─ Verify "AppointmentDB" exists
  ├─ Expand it and verify 4 tables:
  │  ✓ Users
  │  ✓ Doctors
  │  ✓ TimeSlots
  │  ✓ Appointments
  └─ Sample data should be inserted

STEP 3: Test a Query (1 minute)
  ├─ Open new query (Ctrl + N)
  ├─ Type: SELECT * FROM Users;
  ├─ Press F5
  └─ You should see 4 sample users

STEP 4: Connect Your Application (1 minute)
  ├─ Update connection string in your application:
  │  "Server=localhost;Database=AppointmentDB;Trusted_Connection=True;TrustServerCertificate=True"
  ├─ Rebuild and run your application
  └─ Test booking flow end-to-end

YOU'RE DONE! 🎉

💡 KEY CONCEPTS TO REMEMBER
================================================================================

STANDARD SLOTS:
  6 slots per doctor per day (10-11 AM through 4-5 PM, 1 hour each)
  Lunch break: 12-1 PM (no appointments)

USER ROLES:
  "User" → Regular patients who book appointments
  "Admin" → System administrators with full access
  Doctors → Stored in separate Doctors table (not Users table)

APPOINTMENT STATUS:
  "Booked" ................. Normal booking
  "Doctor Unavailable" .... Doctor cancelled the slot
  "Reschedule Suggested" .. Doctor suggested reschedule
  NULL or empty ........... Treated as "Booked"

CRITICAL CONSTRAINT:
  Booking MUST use TRANSACTION to prevent double-booking
  1. Validate slot is available
  2. Insert appointment
  3. Update slot to IsBooked = 1
  4. Commit (all-or-nothing)

FOREIGN KEYS:
  All foreign keys have CASCADE DELETE enabled
  Deleting a doctor deletes all his appointments and slots
  Deleting a user deletes all their appointments

INDEXES:
  Created on frequently searched columns (Email, date, status)
  Composite index on DoctorId + SlotDate for performance
